文件系统

本篇文章一些名词的泛指与 Linux 专用对比（类似）

Linux
文件头 inode

基本组成

linux

  • inode 索引节点 （占用磁盘空间）

    inode 与文件一一对应

  • dentry 目录项（内核维护，缓存在内存，为了提高效率）

    dentry 与 inode 多对一 （？）

dentry 不是目录，目录是文件，dentry 是数据结构，且 dentry 不仅可以表示目录，也能表示文件

 

磁盘格式化分成：

  1. 超级块 （文件系统挂载时进入内存）
  2. 索引节点区 （文件被访问时进入内存）
  3. 数据块区

 

VFS

用户层与文件系统层之间（接口）

linux 支持的文件系统分类：

  • 磁盘
  • 内存 /proc /sys
  • 网络：其他计算机主机

 

文件使用

每个进程维护一个打开文件表，每一项是文件描述符

文件系统的基本操作单位是数据块

（此处存疑？写回磁盘）

 

文件存储

  • 连续

  • 非连续

    • 链表

      • 隐式（无法直接访问数据块）
      • 显式 用于链接文件各数据块的指针，显式地存放在内存的一张链接表（文件分配表 File Allocation Table，FAT）

    • 索引

      每个文件创建一个索引数据块
