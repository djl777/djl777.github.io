树上背包

树上背包1

在点 $u$ 的子树中，选择一个大小恰好为 $m$ 的包含 $u$ 点的连通块，有最大权值和

  • $1 \le n \le 2000$
  • 权值可能为负

 

$f[i][j]$ 表示在 $i$ 这个子树里选择 $j$ 个点，并且包含 $i$ 这个点的连通块的最大权值和

理解成将儿子一个一个地合并上来

每次合并两个背包都要执行一个双重循环，看似是 $O(n^3)$ 的

实际上，两个背包合并时，每对元素只会贡献 $1$ 的代价 $O(size_u \times size_v)$；对于任意一对点，它们只会在 LCA 处被合并，时间复杂度为 $O(n^3)$

 

树上背包2

与上题相同，范围不同

  • $1 \le n \le 50000$
  • $1 \le m \le 100$

显然，合并时只用考虑最多 $m$ 范围内的点对，合并两个子树的代价为 $O(min(size_u, m) \times min(size_v, m))$

总的时间复杂度是 $O(nm)$ 的（难证，记住）

直接 for 到 $m$ 是不对的，变成了 $O(nm^2)$

 

树上背包3

每个点有一个重量 $w_i$ 和权值 $v_i$，选择一个重量和恰好为 $m$ 且包含根的连通块，最大权值和

  • $1 \le n \le 1000$
  • $1 \le m \le 10000$

 

如果按照前两题的思路，由于一个点可能有很大的权值，此时与点数无关，循环需要 for 到 $m$，时间复杂度变成了 $O(nm^2)$

DFS 序上 DP，把问题变成一个特殊的线性问题

求出整棵树的 DFS 序，令 $r_x$ 表示 DFS 序中跳过 $x$ 这个子树的下一个位置

$(1(2(3)(4))(5(6(7))))$，有 $r_2 = 5,r_3 = 4, r_5 = 8$

核心：如果一个点被选进来的话，它的所有父亲都要被选进来

从后往前做，令 $f[i][j]$ 表示考虑 DFS 序中 $[i, n]$ 这一段的节点，选的重量和不超过（题目是等于) $j$ 的点集的最大权值和

对于 $i$ 这个点：

  • 选：$f[i][j] = f[i + 1][j - w_i] + v_i$

  • 不选：$f[i][j] = f[r_i][j]$ （它的所有儿子都要被跳过）

 

树上路径二

与树上路径一相同，但保证每条路径都是从一个点到它的祖先，选择一些路径，使每个点在一条路径上，且权值和最小

 

路径一的两个方法不适用：

  • 一个点可能会在多条路径上面，没有办法直接记录在哪些路径上
  • 直接在每条路径的最高点考虑是否选这个点也比较难转移

对于一个点，如果有很多条路径通过，实际上只用知道路径最多能往上延伸多少

$f(i, j)$ 表示考虑 $i$ 这个子树里面选择的路径，能延伸到深度 $j$（深度越低说明延伸越多）最小权值和

直接做的复杂度会是 $O(n^3)$ 的（$j$ 不是与子树大小成正比的，从一个点出发，多条路径，可以到达上面的多个位置），需要用后缀最小值优化到 $O(n^2)$

$f(i)$：考虑 i 这个子树，i 上已经没有蝴蝶

$g(i)$：考虑 i 这个子树，i 上仍有蝴蝶

$h(i)$：考虑 i 这个子树，i 上有蝴蝶，i 的所有儿子都没有蝴蝶

因此只用考虑 f 的转移

  1. 走到某个儿子时，继续往下探索，则此时其余儿子上的蝴蝶都不可能被抓到，有 $g{v1} + f{v2} + … + f{vn} = \Sigma f_v + a{v1}$，只需要找到最大的儿子
  2. 走到某个儿子即刻返回，则还有机会探索 $t = 3$ 的其余儿子节点，有 $h{v1} + g{v2} + f{v3}… + f{vn} = \Sigma fv + a{v2} - f{v1} + h{v1}$，只需要找到最大的儿子且 $t = 3$，除去 $v_1$ 点，因此需要维护次大值
